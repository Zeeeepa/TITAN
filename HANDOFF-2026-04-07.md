## What Was Shipped (43 commits)

### Agent Core — Tool Calling Pipeline (15 layers)
1. **tool_choice:required** on all rounds in autonomous mode (streaming path)
2. **Input-aware concurrency** — isConcurrencySafe(input) per tool call
3. **Partitioned batching** — groups read-only tools into concurrent batches
4. **Max concurrency cap** — 10 parallel tools
5. **Duplicate detection** — deduplicates same tool+args
6. **SmartExit** — skips to respond after single successful tool call
7. **FabricationGuard** — catches model claiming "I wrote X" without tool calls
8. **IntentParser** — converts "I'll read file X" to read_file(X)
9. **Action Compiler** — parses ACTION: directives to tool calls
10. **ToolRescue** — extracts tool calls from text (JSON, XML, NL patterns)
11. **Self-Heal** — switches models when tool calling fails
12. **Stall Detector** — self-talk loops, analysis-only, no-tool cap, code-in-response
13. **Loop Detection** — circuit breaker, repeat detection, ping-pong
14. **Plan Mode** — plan before acting, user approval gate
15. **Retry with backoff** — 3 retries, exponential backoff, error classification

### Agent Architecture
- **Gemma 4 sampling**: temp=1.0, top_p=0.95, top_k=64 (Google spec)
- **Coding task enforcement**: detects coding requests, injects read→write→test
- **write_file destructive guard**: blocks <40% shrink or >3x expansion
- **edit_file fuzzy matching**: whitespace-normalized + contextual errors
- **append_file tool**: build files incrementally
- **CEO auto-delegation**: breaks tasks into CP issues, delegates to sub-agents
- **Task decomposition**: 4-6 granular subtasks per complex request
- **Context management**: old tool results cleared, head+tail truncation
- **Agent identity prompt**: "You are TITAN, you ACT not describe"
- **Anti-loop prompt rules**: max 1 sentence before calling a tool
- **Hard kill after 2 nudges**: no more infinite retry loops
- **Session compaction thresholds**: 200 runs, 2M tokens, 72h (Paperclip)
